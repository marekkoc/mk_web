🔍 Jak działa $_GET w PHP

KROK 1: Programista pisze HTML z linkami

<a href="?kategoria=jabłka">Jabłka</a>
<a href="?kategoria=banany">Banany</a>
<a href="?kategoria=pomarańcze">Pomarańcze</a>

Uwaga: "kategoria" to dowolna nazwa wymyślona przez programistę!

⬇️

KROK 2: Użytkownik klika link "Jabłka"

Przeglądarka wysyła żądanie HTTP:

GET /strona.php?kategoria=jabłka HTTP/1.1
⬇️

KROK 3: PHP automatycznie analizuje URL

PHP widzi część po "?" i automatycznie tworzy tablicę $_GET:

// PHP robi to ZA CIEBIE:
$_GET = [
    'kategoria' => 'jabłka'
];
⬇️

KROK 4: Programista odczytuje wartość

$kategoria = $_GET['kategoria']; // "jabłka"

Teraz programista może użyć tej wartości!

🧪 Przykład na żywo

Kliknij i obserwuj URL:

🔴 Czerwony | 🔵 Niebieski | 🟢 Zielony | 🏠 Reset

Co PHP widzi:

$_GET['test'] = ""

Wybrany kolor:

$_GET = [] (pusty)

Brak parametrów w URL

📋 Kluczowe fakty

  • "kategoria" to nazwa wymyślona przez programistę (może być cokolwiek!)
  • PHP automatycznie wypełnia $_GET na podstawie URL
  • Użytkownik wypełnia wartości przez kliknięcie linków
  • Format: ?nazwa1=wartość1&nazwa2=wartość2

🔧 Inne przykłady nazw parametrów

?strona=kontakt → $_GET['strona'] = "kontakt"
?id=123 → $_GET['id'] = "123"
?szukaj=php+tutorial → $_GET['szukaj'] = "php tutorial"
?kolor=red&rozmiar=L → $_GET['kolor'] = "red", $_GET['rozmiar'] = "L"
